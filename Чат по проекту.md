# Чат по учебному проекту

Дата: 26 июня 2026

Проект: Панель прораба для контроля ремонта квартиры.

## Идея проекта

Сделать учебный web-прототип, который помогает прорабу показывать заказчику ход ремонта квартиры без постоянного присутствия заказчика на объекте.

Основная мысль: на экране видно, какие этапы ремонта уже завершены, какие идут сейчас, где есть комментарии прораба и замечание заказчика.

## Для кого экран

Экран сделан в первую очередь от лица прораба.

Прораб открывает панель и показывает заказчику:

- общий процент готовности;
- этапы ремонта;
- статус каждого этапа;
- комментарии по работам;
- пример замечания заказчика.

## Этапы ремонта

В первой учебной версии используются этапы:

- демонтаж;
- черновые работы;
- электрика;
- сантехника;
- отделочные работы.

## Состояния экрана

В проекте есть demo-состояния:

- `filled` — обычный заполненный экран;
- `empty` — “Этапы ремонта пока не добавлены”;
- `error` — “Не удалось показать данные этапа”;
- `success` — “Этап завершён”.

## Backend logic map без backend

Пока настоящего backend нет.

Можно использовать только учебные данные:

- название объекта;
- список этапов;
- статусы этапов;
- комментарии прораба;
- одно demo-замечание заказчика;
- выбранное состояние экрана.

Можно временно хранить в `localStorage`:

- выбранное demo-состояние;

- активный этап;
- учебные комментарии.

## Что нельзя собирать сейчас

В учебной версии нельзя собирать:

- реальные адреса квартир;
- ФИО заказчика или прораба;
- телефоны;
- email;
- реальные фото ремонта;
- документы;
- сметы;
- договоры;
- платежи;
- геолокацию;
- пароли;
- API-ключи;
- Telegram-данные.

## Что станет backend только в будущем

Настоящий backend понадобится позже для:

- аккаунтов заказчика и прораба;
- настоящей загрузки фото;
- хранения истории ремонта;
- комментариев и чата;
- уведомлений;
- разных объектов ремонта;
- прав доступа;
- базы данных;
- хранения файлов;
- интеграций с Telegram или CRM.

Сейчас это не добавляется.
