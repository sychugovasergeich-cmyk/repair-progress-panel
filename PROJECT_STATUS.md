# Контроль ремонта квартиры по этапам

## Идея проекта

Учебный web-прототип панели прораба для показа заказчику хода ремонта квартиры.

Прораб открывает страницу объекта и показывает, какие этапы ремонта уже завершены, какие идут сейчас, где есть комментарии и замечания.

## Для кого проект

Главный пользователь первой версии - прораб.

Он использует страницу как красивую демонстрационную панель для заказчика.

## Какую проблему решает

Заказчику не нужно постоянно присутствовать на объекте, чтобы понимать ход ремонта.

Прорабу проще показать прогресс: этапы, статусы, фотоотчет и комментарии собраны в одном месте.

## Учебная первая версия

Одна web-страница без backend и базы данных.

На странице есть:

- название объекта: ремонт квартиры;
- общий процент готовности ремонта;
- этапы ремонта;
- статусы этапов;
- фото-заглушки;
- комментарии прораба;
- пример замечания заказчика;
- панель замечаний со статусами: новое, в работе, закрыто;
- короткий блок "как это работает".

## Этапы ремонта

1. Демонтаж
2. Черновые работы
3. Электрика
4. Сантехника
5. Отделочные работы

## Первый сценарий пользователя

Прораб открывает страницу "Ремонт квартиры" и показывает заказчику:

- общий процент готовности;
- какие этапы завершены;
- какой этап сейчас в работе;
- какие есть комментарии прораба;
- где есть замечание заказчика.
- какие замечания требуют внимания и в каком они статусе.

## Что не делаем сейчас

В первой учебной версии не делаем:

- регистрацию;
- личные кабинеты заказчика и прораба;
- роли пользователей;
- настоящую загрузку фото;
- хранение фото;
- базу данных;
- реальные адреса квартир;
- персональные данные;
- чат;
- уведомления;
- оплату;
- API-ключи;
- Telegram с реальными пользователями;
- production SaaS.

## Почему так

Сейчас цель - не сделать готовый коммерческий продукт, а собрать безопасный учебный прототип.

Он должен показать идею, первый сценарий и структуру будущей панели, но без рисков с данными, оплатами, пользователями и настоящими файлами.

## Следующий маленький шаг

Сделать простую структуру будущей страницы:

1. Верхний блок: название объекта и общий процент готовности.
2. Блок сводки: завершено / в работе / ожидает.
3. Карточки этапов ремонта.
4. Комментарии прораба.
5. Пример замечания заказчика.
6. Блок "как это работает".

## Критерий готовности первого шага

Файл `PROJECT_STATUS.md` создан и по нему понятно:

- что за проект;
- для кого он;
- какую проблему решает;
- что входит в первую версию;
- что мы специально не делаем;
- какой следующий шаг.
